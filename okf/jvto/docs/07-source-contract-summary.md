# Source Contract Summary

## `llm-wiki` contributes candidate knowledge

- Package identifiers, public URL candidates, durations, itinerary headings.
- Policy, trust, destination, FAQ, and guide candidates.
- Source manifests that allow a controlled build gate.

## `jvto-itinerary-core` contributes operational review context

- Route structure and feasibility context.
- Signals that a public itinerary may be misleading or incomplete.
- Potential guide topics requiring public verification.

## Itinerary Core is never auto-published

- Cost components and historical aggregates.
- Recommendation rules.
- Internal operational events.
- Pickup/dropoff patterns.
- Customer-derived data.

Public OKF is a durable knowledge catalog, not an operations data feed.
